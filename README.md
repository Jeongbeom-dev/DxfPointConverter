# DXF 포인트 변환기

AutoCAD DXF 파일을 point-to-point 방식의 **X, Y 포인트**로 변환하는 GUI 프로그램.
(레이저/플로터용 준비 단계 — 현재는 G 명령어 없이 좌표만 출력)

## 실행 방법

### 1) EXE 실행 (Python 불필요 · 단독 실행)
`dist\DxfPointConverter.exe` 더블클릭.
이 파일 **하나만 복사하면 다른 PC(Windows 64bit)에서도 Python 설치 없이 실행**됩니다.
(Python 런타임·tkinter가 모두 내장된 onefile 실행파일)

### 2) 소스 실행
```
python gui.py
```
또는 `run.bat` 더블클릭. (Python 3 + 표준 라이브러리 tkinter 만 필요, 별도 설치 없음)

## EXE 빌드

```
build.bat
```
더블클릭하면 PyInstaller로 `dist\DxfPointConverter.exe`(단독 실행형, 약 12MB)를 생성합니다.
PyInstaller가 없으면 자동 설치합니다.

## 기능

1. **DXF 열기** — DXF(ASCII) 파일 입력
2. **G-code 출력** — 우측 패널에 `G01<탭>X값<탭>Y값` 형식(소수점 3자리)으로 출력. 경로 사이 빈 줄 = 펜 업/이동 구분
3. **헤더 입력** — 출력 상단에 삽입될 헤더를 직접 입력 (예: `G21`, `G90`, 주석 `;` 등)
4. **경로 시각화 (1-layer)** — 좌측 캔버스, 경로별 색상
5. **저장** — `.gcode` 파일로 내보내기 (헤더 + 본문)

### 출력 예시
```
; DXF to G-code
G01	X0.000	Y10.000
G01	X10.000	Y10.000
G01	X10.000	Y5.000
```

## 시각화 조작

| 조작 | 동작 |
|------|------|
| 마우스 휠 | 커서 기준 확대/축소 |
| 드래그 | 화면 이동(팬) |
| 더블클릭 | 화면맞춤 |
| `화면맞춤` / `＋` / `－` | 버튼 줌 |
| `▶ 재생` / `■ 정지` | 그리기 순서 애니메이션 (속도 조절) |

**표시 토글**: 점 · 경로선 · 이동선(펜업, 점선) · 방향(화살표) · 격자 · 시작점(번호)

화면 요소:
- 적응형 **격자** + X/Y 축(강조) + **원점(0,0)** 십자 표시
- **펜업 이동**은 회색 점선으로 구분
- 마우스 위치의 실제 **DXF 좌표**를 우측 하단에 실시간 표시

## 옵션

- **분할 간격** — 원/호/곡선을 일정 구간 길이로 직선 포인트 분할 (작을수록 촘촘·정밀). 값 변경 후 `재변환` 또는 Enter.

## 선분 결합 (폐루프화)

AutoCAD에서 도형을 **여러 개의 개별 LINE/호로 그린 경우**(예: 사각형을 4개의 선으로), 각 선은 열린 경로라 면(폐루프)으로 인식되지 않아 채우기가 되지 않습니다.

- **`선분 결합(폐루프화)`** 체크박스를 켜면, **끝점이 맞닿는 선분들을 자동으로 이어붙여** 하나의 연속 경로로 만들고, 시작점으로 되돌아오면 폐루프로 닫습니다.
- 결합 후에는 면채우기가 정상 동작합니다. 이미 닫힌 폴리라인은 영향 없음.
- 상태바에 `[결합] 외곽선 N(폐루프 M)`으로 결과가 표시됩니다.
- 채우기 시 닫힌 면이 없고 결합으로 해결 가능하면 안내가 뜹니다.

> 참고: AutoCAD `HATCH`(해치/채우기) 엔티티 자체는 아직 미지원이며, 경계를 이루는 LINE/폴리라인을 결합해 사용합니다.

## 면채우기 (슬라이스 / infill)

닫힌 도형(폐루프 = 면)을 일정 간격의 평행선으로 채우는 경로를 생성합니다. (개방 경로는 대상 아님)
개별 선분으로 그려진 도형은 위의 **선분 결합**을 먼저 켜세요.

- **간격** — 채우기 선 사이 간격(mm)
- **방향** — 채우기 선의 각도(0=가로, 45=대각, 90=세로, 135° 등 직접 입력 가능)
- **지그재그** — 켜면 인접한 선을 이어 한 붓 그리기(펜업 최소화), 끄면 개별 선분. 구멍·빈틈·분리된 도형은 자동으로 끊어 빈 공간을 가로지르지 않음
- `채우기 생성` / `채우기 지움`. 외곽선과 색상이 구분되며 `채우기` 토글로 표시 on/off

## 기준점(0,0) 이동

오브젝트 경계 기준으로 원점을 재설정하고 경로/출력을 다시 생성합니다.

| 버튼 | 기준점 |
|------|--------|
| 원본 | DXF 원좌표 유지 |
| 중심 | 경계 중앙 |
| 좌하 / 좌상 | 왼쪽 아래 / 위 모서리 |
| 우하 / 우상 | 오른쪽 아래 / 위 모서리 |

원점(0,0)은 화면에서 빨간 십자로 표시됩니다. 저장되는 포인트 좌표도 선택한 기준점 기준으로 바뀝니다.

## 지원 DXF 엔티티

LINE, LWPOLYLINE(bulge 원호 포함), POLYLINE, CIRCLE, ARC, POINT

## 파일 구성

| 파일 | 설명 |
|------|------|
| `gui.py` | tkinter GUI (메인) |
| `dxf_points.py` | DXF 파싱 + 포인트 변환 모듈 (CLI 단독 실행 가능) |
| `build.bat` | EXE 빌드 스크립트 |
| `run.bat` | 소스 실행 배치 파일 |
| `dist/` | 빌드된 EXE |
| `samples/` | 테스트용 DXF |
| `output/` | 저장 결과 폴더 |

## CLI 단독 사용

```
python dxf_points.py samples/sample.dxf 1.0 > output/points.txt
```
(두 번째 인자 = 분할 간격)
